RAPTOR Y PSEINT HITO — LOGOS IDE
Documento técnico para la integración del depurador didáctico e interactivo Raptor + PSeInt
Plataforma: Logos IDE (Tauri + TypeScript + Ace Editor)

================================================================================
1. PROPÓSITO DEL DOCUMENTO
================================================================================
Este archivo detalla la planeación técnica, la arquitectura y los pasos para implementar un simulador didáctico que fusiona:
- El flujo visual permanente de diagramas inspirado en Raptor, usando el lenguaje propio RE.
- La inspección de código línea a línea del lenguaje propio RE y memoria de variables (inspirado en la didáctica de PSeInt).
Permite ver de forma animada ("física") cómo los valores de las variables origen se leen, se operan en el nodo correspondiente y viajan físicamente hacia la memoria del sistema.

================================================================================
2. COMPORTAMIENTO ESPERADO DE LA INTERFAZ
================================================================================
El panel de depuración (modal de paso a paso) tendrá una vista dividida en 3 columnas fijas:
- COLUMNA IZQUIERDA: Resalta la línea de código del lenguaje RE activa (obtenida directamente del editor de Logos).
- COLUMNA CENTRAL: Muestra permanentemente el diagrama de flujo completo del programa en RE actual. El nodo de la línea activa se ilumina con un brillo y borde distintivos.
- COLUMNA DERECHA (Memoria): Tabla con las variables activas del programa RE y sus valores.

Lógica de Animación Aritmética Dinámica (Ej: int suma = a + b;):
1. Reemplazo Dinámico en el Nodo: El texto del nodo de flujo activo (por ejemplo, "int suma = a + b") cambia a la versión evaluada en tiempo de ejecución ("suma = 5 + 5").
2. Creación de Bloques de Datos: Se instancian dinámicamente dos pequeños bloques HTML sobre el nodo conteniendo los valores de origen (5 y 5).
3. Animación de Fusión: Ambos bloques se deslizan hacia el centro del nodo, aparece un operador "+" intermedio y se fusionan convirtiéndose en un bloque único de valor "10" con un efecto "pop" de escala.
4. Traslación a Memoria: El bloque resultante (10) viaja en una trayectoria curva o recta directamente hasta la celda de la variable "suma" en la columna derecha de variables.
5. Confirmación en Memoria: Al aterrizar el bloque, la celda parpadea en verde/amarillo indicando que el valor ha sido escrito físicamente en la memoria del sistema.

================================================================================
3. ARQUITECTURA DE DATOS (MIGRACIÓN EN STEPPER.TS)
================================================================================
Para soportar esta lógica, la estructura de StepFrame debe expandirse para incluir detalles de resolución de nodos y datos detallados de variables fuente:

interface StepFrame {
  lineNumber: number;         // Línea de código fuente correspondiente
  lineText: string;           // Texto de la línea
  type: string;               // Tipo de instrucción
  variables: Record<string, { value: any; type: string; isNew?: boolean; isChanged?: boolean }>;
  description: string;        // Explicación didáctica
  animationType: string;      // block_sum, block_sub, branch_true, etc.
  animationData: {
    leftVal: any;             // Valor izquierdo evaluado (ej: 5)
    leftName: string;         // Nombre de la variable izquierda (ej: "a")
    rightVal: any;            // Valor derecho evaluado (ej: 5)
    rightName: string;        // Nombre de la variable derecha (ej: "b")
    op: string;               // Operador (ej: "+")
    result: any;              // Resultado obtenido (ej: 10)
    destVar: string;          // Nombre de la variable de destino (ej: "suma")
  };
  nodeId: number;             // ID del nodo del diagrama correspondiente a esta línea
}

4. ARCHIVOS A CREAR O MODIFICAR
================================================================================

4.1 MODIFICAR: Logos-Front/src/stepper.ts
----------------------------------------
- Generación del diagrama dentro del stepper:
  Implementar la función "generateStepperDiagram(code)" que parseará el código y creará una lista de nodos y conexiones visuales (similar a reToBlueprint pero optimizada para renderizarse de forma compacta y sin zoom en el panel central).
- Relación Nodo-Línea:
  Al generar los pasos en "generateSteps(code)", asignar a cada instrucción el ID de su nodo correspondiente.
- Control del Flujo Físico de Animación (Función animateDataFlow):
  Al entrar a un paso de tipo aritmética/asignación:
    1. Localizar mediante getBoundingClientRect() las coordenadas del nodo activo en el lienzo central.
    2. Localizar las coordenadas de la fila de la variable destino en el panel derecho de variables (usando selector [data-varname="suma"]).
    3. Crear un elemento div con clase "floating-block" en el cuerpo de la página.
    4. Aplicar propiedades CSS personalizadas (--startX, --startY, --endX, --endY) para animar el movimiento de traslación desde las coordenadas del nodo hacia la variable.
    5. Escuchar el evento "animationend" del bloque flotante para remover el div y activar la animación de parpadeo ("changed-var") en la fila de variables.

4.2 MODIFICAR: Logos-Front/src/stepper.css
-----------------------------------------
- Estilos del Diagrama Central:
  Añadir reglas para que los nodos del diagrama se posicionen de manera vertical y en escala adaptada al panel central (`.stepper-anim-canvas` debe tener scroll vertical en caso de diagramas largos).
- Animación de Desplazamiento Dinámica:
  Establecer la regla CSS para el bloque flotante:
  ```css
  .floating-block {
      position: fixed;
      z-index: 10000;
      padding: 6px 12px;
      border-radius: 4px;
      background: #b8c3ff;
      color: #13141f;
      font-family: 'Space Mono', monospace;
      font-weight: bold;
      pointer-events: none;

  @keyframes flyToMemory {
      from {
          left: var(--startX);
          top: var(--startY);
          transform: scale(1);
          opacity: 1;
      }
      to {
          left: var(--endX);
          top: var(--endY);
          transform: scale(0.6);
          opacity: 0.3;
      }
  }
  ```
