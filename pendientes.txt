## ESPECIFICACION: AUTOCOMPLETADO DE PALABRAS RESERVADAS DE RE EN ACE EDITOR
### Objetivo
Implementar un completer de Ace Editor que sugiera automáticamente las palabras
reservadas del lenguaje RE cuando el usuario escribe en el editor.
El autocompletado es puramente de keywords — NO requiere análisis semántico.
### Palabras reservadas a autocompletar
  Tipos:         int, double, string, bool, var
  Colecciones:   list, array, queue, stack
  Control flujo: if, else, while, do, for, in, return
  Programa:      program
  I/O / built-ins: print, input, to_str, to_int, to_double, len, size
  Literales:     true, false
  Operadores:    and, or, not
### Implementación técnica en Ace Editor
  Ace usa `ace.require("ace/ext/language_tools")` para habilitar completers.
  Se debe agregar un completer personalizado al array de completers del editor.
  PASO 1 — Agregar el script en index.html (ANTES de index.ts):
    <script src="/ace/ext-language_tools.js"></script>
  PASO 2 — En index.ts, antes de ace.edit(), activar la extensión:
    ace.require("ace/ext/language_tools");
  PASO 3 — Habilitar opciones de autocompletado en editor.setOptions():
    enableBasicAutocompletion: true,
    enableLiveAutocompletion:  true,    // se activa al escribir
    enableSnippets:            true     // permite snippets con Tab
  PASO 4 — Definir array RE_COMPLETIONS con estructura por entrada:
    {
      caption:  "keyword visible en la lista",
      value:    "texto/snippet insertado",
      meta:     "RE",       // etiqueta de categoría
      score:    1000        // prioridad alta (supera al completer de palabras del texto)
    }
  PASO 5 — Snippets recomendados (con $1, $2, ${0} para Tab-stops):
    program  → "program ${1:Main} {\n  ${0}\n}"
    if       → "if (${1:condicion}) {\n  ${0}\n}"
    if/else  → "if (${1:condicion}) {\n  ${0}\n} else {\n  \n}"
    while    → "while (${1:condicion}) {\n  ${0}\n}"
    do-while → "do {\n  ${0}\n} while (${1:condicion});"
    for      → "for ${1:i} in ${2:0..10} {\n  ${0}\n}"
    print    → "print(${1:\"mensaje\"});"
    input    → "string ${1:nombre} = input(\"${2:Prompt: }\");"
    to_str   → "to_str(${1:valor})"
    to_int   → "to_int(${1:valor})"
    to_double→ "to_double(${1:valor})"
    return   → "return ${1:valor};"
  PASO 6 — Registrar el completer (en index.ts, después de editor.setOptions):
    const reCompleter = {
      getCompletions: (_editor, session, pos, _prefix, callback) => {
        const token = session.getTokenAt(pos.row, pos.column);
        // No completar dentro de comentarios o strings
        if (token && (token.type === 'comment' || token.type === 'string')) {
          return callback(null, []);
        }
        callback(null, RE_COMPLETIONS);
      }
    };
    editor.completers = [reCompleter];
    // Si se quiere TAMBIÉN el completer de palabras del buffer actual:
    // const langTools = ace.require("ace/ext/language_tools");
    // editor.completers = [reCompleter, langTools.textCompleter];
### Verificación de archivos necesarios en Logos-Front/ace/
  - ace/ext-language_tools.js   ← requerido para esta feature
  - ace/snippets/text.js        ← requerido si enableSnippets: true
  Copiar desde: node_modules/ace-builds/src-noconflict/ si no existen.
### Consideraciones adicionales
  - La tecla de activación default es Ctrl+Space (puede cambiarse).
  - Con enableLiveAutocompletion: true se dispara automáticamente al escribir.
  - Para reducir el delay: editor.completer puede accederse para configurar timers.
  - Los snippets requieren ace/ext/snippets.js adicional si se usan tab-stops ($1).
