PASO A PASO HITO — LOGOS IDE
Documento técnico para implementación de la funcionalidad "Paso a Paso" (Step-by-step Debugger Visual) Redactado el: 2026-07-15 | Plataforma: Logos IDE (Tauri + TypeScript + Ace Editor)

1. CONTEXTO DEL PROYECTO
El proyecto Logos es un IDE educativo construido con:

Tauri (Rust backend) + TypeScript frontend (sin framework, Vite)
Ace Editor como editor de código principal
Lenguaje propio llamado RE con su intérprete en RE/src/compiler.core/Enviroment/Interpreter.ts
Vista CODE (editor + terminal) y Vista FLOW (diagrama de flujo visual)
Todo vive en Logos-Front/ (frontend) y RE/ (motor del lenguaje)
Archivos clave:

Logos-Front/index.html — estructura HTML de 285 líneas
Logos-Front/src/index.ts — lógica frontend de 2091 líneas
Logos-Front/src/index.css — estilos (~44kb)
RE/src/compiler.core/Enviroment/Interpreter.ts — intérprete RE de 965 líneas
2. QUÉ HAY QUE HACER
Agregar un botón "Paso a Paso" en la barra de herramientas del editor (vista CODE). Al presionarlo, aparece un panel/modal (puede ser lateral o sobre la pantalla) que muestra una animación visual por bloques que simula la ejecución del código línea a línea, visualizando:

Variables enteras (int/double): animación mostrando la suma, resta, multiplicación o división como si fueran bloques que se combinan o separan
Condicionales (if/else): se ve como una bifurcación — el camino que se toma se ilumina
Ciclos (while, for, do-while): se ve como un loop con flecha de retroceso que parpadea cada iteración
Asignaciones: se ve cómo el valor "viaja" de la derecha a la variable
print: aparece una "burbuja de salida" con el valor
Declaraciones: aparece la caja de la variable con su valor inicial
3. ARQUITECTURA DE LA SOLUCIÓN
3.1 Enfoque: Mini-Intérprete de Trazado en el Frontend
NO se debe modificar el intérprete RE (el que corre en Node.js del lado de Rust/CLI). En cambio, se crea un mini-intérprete de trazado en el frontend (TypeScript puro) que:

Lee el código del editor (editor.getValue())
Lo parsea línea por línea con regex (igual que hace syncFlowFromCode en index.ts línea 1639)
Genera un array de "pasos" — cada paso describe qué está pasando
El sistema de animación consume ese array de pasos uno por uno, con botones Anterior/Siguiente/Play
3.2 Estructura de un "Paso"
typescript


interface StepFrame {
  lineNumber: number;         // línea del código fuente
  lineText: string;           // texto de esa línea
  type: 'declare' | 'assign' | 'arithmetic' | 'condition' | 'loop' | 'print' | 'input' | 'return' | 'program_start' | 'program_end';
  variables: Record<string, { value: any; type: string; isNew?: boolean; isChanged?: boolean }>;
  description: string;        // texto educativo: "Se declara la variable x con valor 5"
  animationType: 'block_sum' | 'block_sub' | 'block_mul' | 'block_div' | 'branch_true' | 'branch_false' | 'loop_enter' | 'loop_back' | 'assign_flow' | 'print_bubble' | 'none';
  animationData?: any;        // datos específicos para la animación (operandos, resultado, etc.)
  activeVariable?: string;    // variable que está cambiando
  conditionResult?: boolean;  // resultado de la condición (para if/while)
  loopIteration?: number;     // iteración actual del ciclo
}
4. ARCHIVOS A CREAR / MODIFICAR
4.1 ARCHIVO NUEVO: Logos-Front/src/stepper.ts
Este archivo contiene TODO el motor del paso a paso. Debe exportar:

typescript


export function generateSteps(code: string): StepFrame[]
export function openStepper(steps: StepFrame[]): void
export function closeStepper(): void
Función generateSteps(code):

Usa un mini-intérprete con regex (NO ANTLR, NO Node)
Recorre el código línea por línea
Mantiene un objeto variables: {} que va actualizando
Por cada línea relevante, genera un StepFrame
Evalúa expresiones simples con eval() o manualmente
Patrones regex a detectar (igual que syncFlowFromCode):



/^\s*program\s+(\w+)\s*\{/           → StepFrame type='program_start'
/^\s*(int|double|string|bool)\s+(\w+)\s*=\s*(.+?)\s*;/  → type='declare'
/^\s*(\w+)\s*([\+\-\*\/]?=)\s*(.+?)\s*;/                → type='assign'
/^\s*if\s*\((.+?)\)\s*\{/            → type='condition'
/^\s*\}\s*else\s*\{/                 → (else branch)
/^\s*while\s*\((.+?)\)\s*\{/         → type='loop'
/^\s*for\s+(\w+)\s+in\s+(.+?)\s*\{/ → type='loop'
/^\s*do\s*\{/                         → type='loop'
/^\s*print\s*\((.+)\)\s*;/           → type='print'
/^\s*\}/                              → cierre de bloque
Evaluación de expresiones aritméticas:

Para int x = a + b, extraer operandos y operador
Guardar en animationData: { left: a, op: '+', right: b, result: a+b }
Esto permite animar "bloque a + bloque b = bloque resultado"
Simulación de ciclos:

Para for i in 0..10, generar un StepFrame por cada iteración con loopIteration: i
Para while (cond), simular máximo 20 iteraciones (límite educativo)
Cada iteración del ciclo genera sus propios StepFrames internos

5. LÓGICA DETALLADA DE ANIMACIONES
5.1 Animación de ARITMÉTICA (int/double)
Cuando se detecta int x = a + b o x += 5:

Estructura del HTML generado dinámicamente en #stepperAnimCanvas:

[BLOQUE_A: valor_a] [OPERADOR: +] [BLOQUE_B: valor_b] → [BLOQUE_RESULTADO: resultado]
Animación CSS:

Los bloques A y B aparecen desde los costados (slide-in)
El operador aparece en el centro con fade-in
Ambos bloques se mueven hacia el centro (transform: translateX)
Desaparecen y aparece el bloque resultado con un "pop" (scale: 0 → 1.2 → 1)
El resultado vuela hacia la caja de la variable en la tabla derecha
Para suma: bloques se juntan → se fusionan Para resta: el bloque derecho "sale" del izquierdo Para multiplicación: el bloque se replica visualmente (clonar con CSS) Para división: el bloque se parte en dos mitades

5.2 Animación de CONDICIONAL (if/else)
Cuando se detecta if (condicion):



     [CONDICION]
         |
    [¿ES TRUE?]
    /          \
[SI → TRUE]  [NO → FALSE]
Animación:

Aparece un rombo animado (el mismo estilo que el Flow View)
La condición se evalúa y se muestra el resultado (TRUE/FALSE)
El camino que se toma se ilumina con un pulso de luz (box-shadow animado)
El camino NO tomado se oscurece (opacity: 0.2)
5.3 Animación de CICLOS (while, for)
Cuando se detecta un bucle:

Visualización:

Aparece un bloque de ciclo con flecha circular (↻)
Una barra de progreso muestra "Iteración 1 de N"
Cada iteración hace que la flecha gire 360° con animación CSS
Los pasos DENTRO del ciclo se ejecutan normalmente en cada iteración
Al final del ciclo, la flecha se rompe (desaparece con un fade-out)
5.4 Animación de ASIGNACIÓN
Cuando x = valor:

El valor aparece a la derecha como un bloque
Una flecha animada (→) lo "empuja" hacia la caja de la variable
La caja de la variable en la tabla parpadea en verde momentáneamente
5.5 Animación de PRINT
Cuando print("texto") o print(variable):

Aparece una burbuja de diálogo (estilo chat bubble) desde abajo
Dentro de la burbuja aparece el valor que se imprime
La burbuja asciende y desaparece con fade-out
6. COMPONENTE DE CÓDIGO RESALTADO
En #stepperCodeView, mostrar el código del editor con:

Todas las líneas visibles (scroll si es largo)
La línea actual resaltada con fondo #b8c3ff20 y borde izquierdo de 3px #b8c3ff
Al avanzar pasos, la línea resaltada cambia con transición suave
Las líneas ya ejecutadas tienen un tono más apagado (opacity: 0.5)
Las líneas futuras tienen tono normal
Implementación:

typescript


function renderCodeView(steps: StepFrame[], currentIndex: number) {
    const codeLines = editor.getValue().split('\n');
    const currentLine = steps[currentIndex].lineNumber; // 1-based
    
    stepperCodeView.innerHTML = codeLines.map((line, i) => {
        const lineNum = i + 1;
        const isPast = lineNum < currentLine;
        const isCurrent = lineNum === currentLine;
        const cls = isCurrent ? 'step-line-current' : isPast ? 'step-line-past' : 'step-line-future';
        return `<div class="step-code-line ${cls}">
            <span class="step-line-num">${lineNum}</span>
            <span class="step-line-text">${escapeHtml(line)}</span>
        </div>`;
    }).join('');
    
    // Scroll automático a la línea actual
    stepperCodeView.querySelector('.step-line-current')?.scrollIntoView({ block: 'center', behavior: 'smooth' });
}

8. CONTROLES DE PLAYBACK
Botón	Función
⏮	Ir al primer paso
◀ Anterior	Retroceder un paso
▶ Play / ⏸ Pause	Reproducción automática
Siguiente ▶	Avanzar un paso
⏭	Ir al último paso
Selector velocidad	Lento (2s) / Normal (1s) / Rápido (0.4s)
Play automático: usa setInterval con el tiempo seleccionado. Al llegar al último paso, se detiene y muestra "¡Ejecución completada!".

10. FLUJO DE EJECUCIÓN COMPLETO (para la IA implementadora)

        ↓

        ↓

        ↓
[Array de StepFrames]
        ↓
openStepper(steps, editor)
        ↓
Renderiza #stepperOverlay (display: flex)

Muestra paso 0
        ↓
Usuario navega con botones o Play
        ↓
  goToStep(index):
    1. renderCodeView(index) → resalta línea
    2. renderAnimation(step) → construye HTML de animación
    3. renderVarTable(step.variables) → actualiza tabla
    4. renderDescription(step.description) → texto educativo
    5. updateProgress(index, total) → barra de progreso
        ↓
Play mode: setInterval → goToStep(current++) hasta llegar al final
        ↓
Usuario cierra → closeStepper() → display: none, limpiar estado
11. CASOS ESPECIALES Y LÍMITES
Código con errores: Si generateSteps no puede parsear, retorna [] y se muestra un toast de error.
Ciclos infinitos: Máximo 50 iteraciones simuladas. Mostrar advertencia: "⚠ Ciclo limitado a 50 iteraciones por razones educativas".
Funciones definidas por el usuario: En primera versión, mostrar como un bloque "FUNCIÓN: nombre()" sin entrar dentro. Futuras versiones pueden expandir.
Expresiones complejas: Si la expresión tiene más de 2 operandos, evaluar con Function() de forma segura y mostrar solo resultado.
Variables no declaradas: Ignorar el paso sin error.
Código vacío o solo comentarios: Mostrar toast "No hay pasos que mostrar".
